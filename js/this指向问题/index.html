This指向问题

this指向问题以及如何改变

指向问题

  1. 一般函数中this指向window。
  2. 对象中函数的this指向，他的this是调用该函数的实例对象
  3. 构造函数里的this，指向创建出来的实例
  4. 事件处理函数中，他的this指向触发事件的元素
  5. 箭头函数没有自己的this，它的this指向上下文中的this，即所处环境的this
   函数名();  ==>  属于函数调用模式，函数内的this指向window
   new Person(); ==> 构造函数中的this指向了新创建的实例对象
   方法调用模式 （对象中的函数称之为方法）==> obj.sayHi()  ==> 对象.方法名() ==> 方法内的this指向了调用方法的哪个对象(谁调用指向谁)
   上下文调用 ==> 可以修改ithis的指向

修改指向

使用call、apply、bind方法

这三个方法存在于Function.prototype(原型对象)

函数都可以使用这三个方法实现修改this的指向

call()

调用函数可以使用()来调用，除此之外，还可以使用call方法来调用函数

   函数名().call(this指向)

call的第一个参数就是用来修改函数的this指向的

除此之外，其他所有的参数都是用来给函数传递实参的

如果call的第一个参数是null或者undefined，或则和不写函数内的this指向window。

apply()

call 和 apply() 的作用

​ 1.调用函数

​ 2.第一个参数修改this指向

​ 3.还可以给函数传递实参

call(修改this指向，参数1，参数2，……)

apply(修改this指向，[参数1，参数2，……])

​ apply具有平铺性，数组中的每一项取出来给函数传递实参

bind

语法：var fn2 = fn.bind(thisArg);

作用：创建并返回函数，新函数和fn函数长得一样，新函数的this指向bind的thisArg函数

总结：

call 和 apply 作用一样

​ 1.能调用函数

​ 2.第一个参数能修改this指向

​ 3.给函数传递实参

bind 是创建并返回新函数，不调用函数
