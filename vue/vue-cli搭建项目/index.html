Vue Cli搭建项目

通过vue-cli搭建项目

  • 在需要创建的文件夹下，按住shift键右击打开Powershell窗口
  • 输入指令创建项目（project)
vue create project
  • 使用git管理项目
git init
git add .
git commit -m '初始化项目'
  • 登录github创建一个空的远程仓库
  • 给仓库设置别名，并将本地代码推送到远程仓库
git remote add origin git@github.com:AchaH/project.git
git push -u origin master

  • 清理脚手架默认生成的内容

    • 清理APP.vue
    • 修改router/index.js
    • 把views components assets文件夹都清空

  • 通用样式的处理

    • 新建styles文件夹创建新的文件base.less
    * {
  margin: 0;
  padding: 0;
  list-style: none;
  box-sizing: border-box;
}
    • 如果有icon图标，引入iconfont.css
    • 在main.js导入这两个样式文件
    // 导入通用的样式
import './styles/base.less'
import './styles/iconfont.css'

  • 引入vant组件库

    • 安装vant-ui
    yarn add vant
    • 全局导入(方便)
    import Vant from 'vant';
import 'vant/lib/index.css';
  
Vue.use(Vant);
    • 按需加载(性能高)
    1. 安装插件
npm i babel-plugin-import -D
  
2. 在babel.config.js中
{
  "plugins": [
    ["import", {
      "libraryName": "vant",
      "libraryDirectory": "es",
      "style": true
    }]
  ]
}
  
3. 在main.js中导入某个组件
import Vue from 'vue';
import { Button } from 'vant';
  
Vue.use(Button);

  • 配置项目中的rem

    amfe-flexible库

    • 安装
    yarn add amfe-flexible
    • 在main.js中引入
    import 'amfe-flexible'
    • 效果
    html的fontSize会被设置为屏幕宽度的 1/10

    postcss-pxtorem

    • 安装依赖包
    yarn add postcss-pxtorem -D
    • 提供postcss的配置文件postcss.config.js
    module.exports = {
  plugins: {
    // autoprefixer: {
    //   browsers: ['Android >= 4.0', 'iOS >= 8']
    // },
    'postcss-pxtorem': {
      // 1rem的大小
      // 1rem大小需要设置为37.5  并且量出来的大小也要 /2
      rootValue: 37.5,
      // 把哪些属性的px转成rem
      propList: ['*']
    }
  }
}

至此，基本配置完成，可以开始写项目

关于配置eslint和prettier

确保代码规范

eslint仅仅对js进行格式化和校验，不会格式化html和css

  • 安装eslint插件

     在setting.json中配置
 // 当保存的时候，eslint自动帮我们修复错误
  "editor.codeActionsOnSave": {
    "source.fixAll": true
  },

  • prettier插件 格式化代码（html/js/css/json)

    1. 安装prettier插件
  
2. 再settings.json文件中增加一个配置
"editor.formatOnSave": true

  • 解决eslint和prettier冲突问题

    • 在项目中提供 prettier的配置文件.prettierrc
    {
  "semi": false,
  "singleQuote": true
}
    • 如何修改eslint的规范, 在.eslintrc配置文件中修改eslint的配置
      rules: {
    'no-console': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
    'no-debugger': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
    // key: 规则的名字   off: 关闭这条规则
    'space-before-function-paren': ['error', 'never']
  }
