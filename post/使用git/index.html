使用git

Mon, Sep 21, 2020 2-minute read

关于git

git 使用

  1. 创建一个空目录

    在想要存放的文件夹下面执行 mkdir 文件夹名称

  2. 在命令行中进入到我们的项目目录

    cd 文件夹名称

  3. git仓库初始化（让git知道，他需要来管理这个目录）

    git init 会在项目目录下生成一个隐藏文件夹.git ，不能删除，也不能随意更改

git 常用指令操作

查看当前状态：git status

红色表示工作区中的文件需要提交

绿色表示暂存区中的文件需要提交

添加到缓存区：git add

git add 指令可以添加一个文件，也可以添加多个文件。
语法1：git add 文件名
语法2 ：git add 文件1 文件2 文件3
语法3： git add .  提交工作区所有的文件到暂存区（添加当前目录到缓存区中）

提交至版本库： git commit -m ‘提交说明’

版本回退

版本回退分为两个步骤：

​ 1、查看版本，确定需要回到的时刻点

​ 指令：git log

​ git log –pretty=online (较推荐)

​ 2、回退操作

​ 指令： git reset –hard 版本号（即commit id）

注意：

回到过去之后想要再回到之前最新的版本的时候，需要使用指令区查看历史操作，以得到最新的commit id

指令： git reflog （记录了你的每一次命令）

小结：

  • 要想回到过去，必须先得到 commit id,然后通过 git reset –hard 进行回退
  • 要想回到未来，需要使用git reflog 进行历史操作查看，得到最新的 commit id
  • 在写回退指令的时候commit id 可以不用写全，git会自动识别，但是也不能写太少，至少需要写前四个字符。

操作仓库

在仓库上做对应的操作（提交暂存区、提交本地仓库、提交线上仓库、拉取线上仓库）

提交到线上仓库的指令： git push

在首次往线上仓库push提交内容的时候出现403的知名错误，原因是不是任何人都可以往线上仓库提交内容，必须需鉴权。

需要修改‘.git/config’文件中的url路径

将url=https://github.com/用户名/仓库名.git

修改为：

url=https://用户名：密码@github.com/用户名/仓库名.git

拉取线上仓库：git pull

基于SSH协议

步骤：

使用ssh-keygen-t rsa命令生成一堆密钥（公钥(id _res .pub)和私钥）

将公钥上传到远程仓库进行配置

ps:现在本地项目使用私钥进行加密，再上传到远程仓库使用公钥进行解密

分支相关指令

查看分支：git branch

  • 在当前分支的前面会有一个*

创建分支：git branch 分支名

切换分支： git checkout 分支名

创建并切换分支： git checkout -b 分支名

删除分支：git branch -d 侧分支名称

  • 不能再当前分支删除当前分支，需要切换到其他分支才能删除

合并分支：git merge 侧分支名称

  • 在侧分支修改之后重新进行 add 和 commit 操作之后再切换回主分支，将侧分支的内容与master主分支合并。

合并所有分支之后需要将master分支上的内容git push到线上远程仓库中

git 冲突与解决

产生冲突：

本地仓库和线上远程仓库的内容不一致。

解决冲突：

  • 先git pull （此时git 已经将线上与本地仓库的冲突合并到了对应的文件夹中）

  • 打开冲突文件，解决冲突

    解决：需要和之前提交的人进行商量，看代码如何调整，保留需要的，删除不需要的

  • 重新提交（add， commit， push）

git实用工具

  • Github for Desktop
  • Source tree
  • TortoiseGit

忽略文件

新建一个名为.gitignore的文件，该文件用于声明忽略文件或不忽略文件的规则，规则对当前目录及其子目录生效

注意：该文件因为没有文件名，没办法直接再window 目录下直接创建，可以通过命令 Git Bash来touch 创建、

常见规则写法：

  1. /mtk/ 过滤整个文件夹

  2. *.zip 过滤所有.zip文件

  3. /mtk/do.c 过滤某个具体文件

  4. !index.php 不过滤具体某个文件

    注意：在文件中以#开头的都是注释

指令：

touch .gitignore

编写文件中的规则：比如忽略掉js目录

# 忽略掉/js目录
/js/

配置密钥对

使用ssh-keygen-t rsa命令生成一堆密钥（公钥和私钥）

将公钥上传到远程仓库进行配置

现在本地项目使用私钥进行加密，再上传到远程仓库使用公钥进行解密

大事件项目

git使用注意：多个人同时开发一个项目，每次上传到仓库之前需要git pull拉取一下，避免冲突。

submit和button会自动触发form表单的默认请求方式get，需要请求之前阻止默认行为

导入静态资源

  1. git clone 远程仓库

  2. cd 远程仓库项目名称，tab键可以自动补齐

  3. 此时进入到项目的主分支

  4. 创建并切换分支

    git checkout -b 分支名称

  5. 将静态资源复制粘贴到你的当前项目的文件夹中

  6. git status 查看当前文件的状态

    红色表示工作区中的文件需要提交

    绿色表示暂存区中的文件需要提交

  7. git add . 提交工作区的所有文件到暂存区

  8. git commit -m ‘提交说明’ 提交暂存区的所有文件到仓库区

  9. git checkout master 回到主分支

  10. git merge 分支名称 将其他分支的内容合并到当前分支，由于当前处于主分支，于是把侧分支合并到主分支

  11. git branch -d 分支名称 删除侧分支，不能在当前分支删除当前分支，需要切换到其他分支才能操作删除

  12. git push 仓库地址 master 将本地仓库中代码提交到远程仓库，注意：master 分支必须写，不能省略
